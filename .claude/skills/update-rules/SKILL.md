今回の会話で新たに得た知見・決定・規約を適切なファイルに追記・更新する。

## 対象ファイルと記載内容の基準

| ファイル | 記載すべき内容 |
|---|---|
| `CLAUDE.md` | プロジェクト概要・アーキテクチャ・環境変数・開発コマンドの変更 |
| `.claude/rules/coding.md` | import規則・命名規則・Biome設定・型安全性に関するルール |
| `.claude/rules/patterns.md` | 実装パターン・ユーティリティ関数の使い方・エラーハンドリング方針 |
| `.claude/rules/infra.md` | インフラ・CI/CD・デプロイ・シークレット管理に関する知見 |

## 手順

1. **抽出**
   $ARGUMENTS が指定されていればそれを記録対象とする。
   指定がなければ今回の会話を振り返り、以下に該当するものを抽出する：
   - 新たに決定したルールや規約
   - 既存の実装から読み取れた重要なパターン（まだrulesに書かれていないもの）
   - 今後の作業で参照すべき設計判断・制約
   - skillsやツールの使い方に関する学び（`.claude/` 配下のルールとして記録）

   「当たり前のこと」「一時的な判断」「タスク固有の話」は記録しない。

2. **分類**
   各抽出項目を上記の表に従いどのファイルに書くか決める。
   複数ファイルにまたがる場合はそれぞれに記載する。

3. **重複チェック**
   対象ファイルを Read して既存の記述と重複・矛盾がないか確認する。
   既存の記述と矛盾する場合は上書き修正する。重複は追記しない。

4. **更新**
   Edit で各ファイルに追記・修正する。
   - 既存のセクションに自然に溶け込む形で書く
   - 新しいセクションが必要な場合は `##` 見出しを追加する
   - 箇条書きは簡潔に（1項目2行以内を目安）

5. **報告**
   更新したファイルと追加・変更した内容を箇条書きで報告する。
   更新対象がなければ「記録すべき新しい知見はありませんでした」と伝える。
